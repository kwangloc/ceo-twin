# Executive Communication Skill

Use this skill whenever communicating information to people (Slack, email, chat messages, meeting notes, updates, reminders, announcements, status reports).

## Goal
Ensure messages are concise while preserving all critical operational details required for action or decision-making.

## Rules
- Use clear subject lines — Make it easy for recipients to understand the topic and urgency of the message.
- Use Vietnameses to communicate
- State key facts explicitly — Never assume recipients know previously discussed information.
- Include dates and deadlines — If a due date, milestone, meeting time, or deadline is known and relevant, state it explicitly.
- Include ownership — Mention the owner, team, or responsible party when relevant.
- Include required actions — Clearly state what recipients need to do, if anything.
- Avoid ambiguity — Replace pronouns such as "it", "that", "this task", or "the project" with the actual project or deliverable name when clarity matters.
- Be concise — Prefer 2–5 short sentences or a small bullet list.
- Preserve important facts — Never omit retrieved facts, decisions, dates, risks, blockers, or action items for the sake of brevity.

## Example
- Bad: 
```
Reminder that Neo4j staging deployment is coming up soon.
```
- Good: 
```
Reminder: Neo4j staging deployment is scheduled for 2024-06-15 at 10:00 AM UTC. Please ensure all code changes are merged by 2024-06-14 COB. Contact @devops-team for any issues.
```
