# Task Decomposition Skill

Use this skill to break a complex request into actionable steps with clear dependencies.

## Process

1. **Clarify the goal** — What does "done" look like?
2. **Identify sub-tasks** — What discrete pieces of work are required?
3. **Map dependencies** — Which tasks must complete before others can start?
4. **Assign ownership hints** — Who is best placed to own each sub-task?
5. **Sequence the work** — Propose an execution order that respects dependencies.

## Output Format

### Goal
[Clear statement of the desired outcome]

### Sub-tasks

| # | Task | Owner | Depends On |
|---|------|-------|------------|
| 1 | …    | …     | —          |
| 2 | …    | …     | Task 1     |

### Recommended Execution Order
1. [Task N]
2. [Task M]

### Risks and Open Questions
- [Risk / question]
